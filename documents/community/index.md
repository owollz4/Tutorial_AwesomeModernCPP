# 社区文章

这里收录 Tutorial_AwesomeModernCPP 社区贡献的文章、笔记、源码阅读、工程经验和高质量问答整理。

社区文章不强制进入主线教程卷。它提供一个更开放的入口：投稿者可以先提交 Markdown，维护者做基础检查后上线展示，再根据讨论和审阅结果决定是否长期收录，或进一步整理进主线章节。

## 流转方式

1. 投稿者提交 Markdown 文件。
2. 维护者检查基础质量、版权来源和明显技术错误。
3. 通过基础检查后，文章进入 `community/incoming/`，并可在文档站和 TAMCPP 周报中展示。
4. 文章经过社区讨论、语法修订和技术审阅后，移动到 `community/articles/`。
5. 如果文章非常适合主线教程，维护者可进一步整理进对应卷或章节。

## 投稿边界

投稿者可以只负责正文内容，不需要一开始理解完整站点结构。

建议提供：

- 文章标题和作者署名。
- 正文 Markdown。
- 图片、代码和引用资料的来源说明。
- 目标读者或适用背景。
- 是否允许维护者调整标题、格式、放置位置和部分措辞。

维护者负责：

- 判断文章放入初刊区、收录区还是主线教程。
- 补齐必要的 frontmatter、导航、索引和链接。
- 做基础格式整理、术语统一和技术审阅。
- 判断是否需要英文翻译或后续专题化整理。

## 最低收录要求

社区初刊不是最终定稿，但上线前仍需满足基本要求：

- 内容可以正常渲染。
- 没有明显技术硬伤。
- 原创或已获得明确授权。
- 引用外部资料时给出来源。
- 图片、代码和大段材料来源清楚。
- 作者同意公开展示，并允许维护者进行必要编辑。

学习问题、路线讨论和开放式建议请优先使用 GitHub Discussions；明确的内容提案或投稿主题可以使用 GitHub Issue。

项目自身的维护节奏、站点迭代和发布度量记录在 [项目开发](dev/) 中。
