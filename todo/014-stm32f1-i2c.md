# STM32F1 I2C 完整教程

## 目标
编写 STM32F1 I2C 外设的完整教程，覆盖 I2C 协议基础（SCL/SDA 双线、开漏输出与上拉电阻、7/10 位地址机制、ACK/NACK 应答、起始/停止条件）、HAL I2C 读写 API、I2C 设备 ID 读取、传感器寄存器读取。使用 C++23 特性（std::span、std::expected、std::to_underlying）进行类型安全的驱动封装。教程遵循 9 节模板结构。

## 验收标准
- [ ] 教程遵循 9 节模板：目标 -> 硬件原理 -> HAL 接口 -> 最小 Demo -> C++23 封装 -> 常见坑 -> 练习题 -> 可复用代码 -> 小结
- [ ] 硬件原理部分包含 I2C 通信时序图（起始条件、地址帧、ACK/NACK、数据帧、停止条件）
- [ ] 解释开漏输出与上拉电阻的硬件设计要点（计算上拉电阻值：Rp = (Vcc - 0.4V) / 3mA）
- [ ] HAL 接口覆盖：`HAL_I2C_Master_Transmit`、`HAL_I2C_Master_Receive`、`HAL_I2C_Mem_Write`、`HAL_I2C_Mem_Read`、中断/DMA 版本
- [ ] 最小 Demo 包含：I2C 总线扫描（扫描 0x03-0x77 地址）、传感器 ID 读取、BMP280 温压传感器读取
- [ ] C++23 封装：`I2cDriver` 类 + `I2cDevice` 抽象，使用 `std::to_underlying` 处理寄存器地址枚举
- [ ] 常见坑涵盖：I2C 总线锁死（SCL 被拉低的恢复方法）、时钟拉伸、多主机冲突
- [ ] 练习题包含：BMP280 完整驱动、EEPROM 读写、多传感器管理
- [ ] 所有代码在 STM32F103C8T6 + BMP280 模块上验证

## 实施说明
### 9 节模板详细规划

**第 1 节：目标**
- 理解 I2C 协议双线通信原理与地址机制
- 掌握 STM32F1 I2C 外设的 HAL 驱动 API
- 能用 C++23 封装通用的 I2C 驱动和传感器抽象层

**第 2 节：硬件原理**
- I2C 双线总线：SCL（时钟，由主机驱动）、SDA（数据，双向）
- 开漏输出 + 上拉电阻的硬件设计原理
- 上拉电阻选型计算：标准模式（100kHz）4.7kΩ、快速模式（400kHz）2.2kΩ
- 通信时序详解：
  - 起始条件（S）：SCL 高电平时 SDA 从高到低
  - 停止条件（P）：SCL 高电平时 SDA 从低到高
  - 重复起始条件（Sr）：不释放总线的再次起始
  - 地址帧：7 位地址 + R/W 位 + ACK/NACK
  - 数据帧：8 位数据 + ACK/NACK
- 7 位地址 vs 10 位地址
- STM32F1 I2C 外设：I2C1（APB1, 36MHz）、I2C2（APB1, 36MHz）
- GPIO 配置：SCL/SDA 复用开漏输出（需外接上拉电阻）
- 硬件连接图：Blue Pill + BMP280 模块接线图（注意上拉电阻）

**第 3 节：HAL 接口**
- `I2C_HandleTypeDef` 初始化结构体详解（ClockSpeed、OwnAddress1、AddressingMode、DualAddressMode、GeneralCallMode、NoStretchMode）
- 标准模式（100kHz）vs 快速模式（400kHz）的时钟配置
- 主机 API：
  - `HAL_I2C_Master_Transmit(huart, DevAddress, pData, Size, Timeout)`
  - `HAL_I2C_Master_Receive(huart, DevAddress, pData, Size, Timeout)`
- 寄存器读写 API（最常用）：
  - `HAL_I2C_Mem_Write(huart, DevAddress, MemAddress, MemAddSize, pData, Size, Timeout)`
  - `HAL_I2C_Mem_Read(huart, DevAddress, MemAddress, MemAddSize, pData, Size, Timeout)`
- 中断/DMA 版本：`HAL_I2C_Master_Transmit_IT`、`HAL_I2C_Mem_Read_DMA` 等
- 回调函数：`HAL_I2C_MasterTxCpltCallback`、`HAL_I2C_MasterRxCpltCallback` 等
- I2C 总线恢复：`HAL_I2C_DeInit` + 手动时钟脉冲恢复

**第 4 节：最小 Demo**
- Demo 1：I2C 总线扫描器（遍历 0x03-0x77，尝试 `HAL_I2C_IsDeviceReady`，打印发现的设备地址）
- Demo 2：读取 BMP280 的 ID 寄存器（寄存器 0xD0，期望返回 0x58）
- Demo 3：读取 BMP280 校准参数并计算温度值
- Demo 4：完整的温压数据采集并通过 UART 打印

**第 5 节：C++23 封装**
- `I2cDriver` 类：底层 I2C 主机封装
  ```cpp
  class I2cDriver {
  public:
      auto write(uint8_t dev_addr, std::span<const std::byte> data, uint32_t timeout_ms) -> std::expected<void, I2cError>;
      auto read(uint8_t dev_addr, std::span<std::byte> buffer, uint32_t timeout_ms) -> std::expected<void, I2cError>;
      auto write_register(uint8_t dev_addr, uint8_t reg, std::span<const std::byte> data, uint32_t timeout_ms) -> std::expected<void, I2cError>;
      auto read_register(uint8_t dev_addr, uint8_t reg, std::span<std::byte> buffer, uint32_t timeout_ms) -> std::expected<void, I2cError>;
      auto scan() -> std::expected<std::vector<uint8_t>, I2cError>;
      auto is_device_ready(uint8_t dev_addr) -> bool;
  };
  ```
- `I2cDevice` 抽象类：传感器/设备通用基类
  ```cpp
  class I2cDevice {
  public:
      constexpr I2cDevice(I2cDriver& bus, uint8_t address) : bus_(bus), address_(address) {}
      auto read_id() -> std::expected<uint8_t, I2cError>;
      virtual auto initialize() -> std::expected<void, I2cError> = 0;
  protected:
      auto write_reg(uint8_t reg, std::span<const std::byte> data) -> std::expected<void, I2cError>;
      auto read_reg(uint8_t reg, std::span<std::byte> buffer) -> std::expected<void, I2cError>;
  };
  ```
- 使用 `std::to_underlying` 将寄存器地址的 `enum class` 转为底层整数
- 寄存器地址的 `constexpr enum class` 设计模式

**第 6 节：常见坑**
- I2C 总线锁死：从设备拉低 SDA 不释放，需发送 9 个额外时钟脉冲恢复
- STM32F1 I2C 的已知 BUG（ERRATA）：BUSY 标志卡死，需软件复位 I2C 外设
- 时钟拉伸（Clock Stretching）：某些从设备会拉低 SCL，需启用 NoStretchMode 或等待
- 多次 `HAL_I2C_Master_Transmit` 之间的时序要求
- BMP280 上电后需要等待启动时间才能读取
- I2C 地址左移问题：HAL 函数期望 7 位地址左移 1 位（即 0xEE 而非 0x77）

**第 7 节：练习题**
- 基础：实现通用的 I2C EEPROM（AT24C02）读写驱动
- 进阶：实现 BMP280 完整驱动（包含校准参数读取、温度/气压补偿计算）
- 挑战：实现一个 I2C 设备管理器，支持动态注册多个设备并统一管理初始化和错误处理

**第 8 节：可复用代码**
- `i2c_driver.hpp` / `i2c_driver.cpp`：I2cDriver 完整实现
- `i2c_device.hpp`：I2cDevice 抽象基类
- `bmp280.hpp` / `bmp280.cpp`：BMP280 驱动示例（含校准计算）
- `i2c_scanner.hpp`：通用 I2C 总线扫描工具

**第 9 节：小结**
- I2C 协议核心概念回顾
- 与 SPI 的对比选型指南
- C++23 封装的寄存器地址 enum class 模式总结
